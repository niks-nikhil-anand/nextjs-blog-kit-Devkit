MDX is a powerful way to write content that blends Markdown with interactive React components. For developers building a blog or documentation, it's the gold standard. In this guide, we'll build a custom MDX pipeline using the Next.js 14 App Router and `next-mdx-remote`.

## Why Build Your Own Pipeline?

While tools like Contentlayer are great, they can sometimes be heavy or fall out of maintenance. Building your own pipeline gives you total control over how files are read, how metadata is parsed, and how components are rendered.

## Step 1: Install Dependencies

We'll need a few packages to handle frontmatter parsing, reading time, and syntax highlighting.

```bash
npm install next-mdx-remote gray-matter reading-time shiki rehype-pretty-code rehype-slug
```

## Step 2: Reading MDX Files

We need a utility to read files from the `content/posts` directory and parse the frontmatter. We'll use `gray-matter` for this.

```typescript title="lib/mdx.ts"
import fs from "fs";
import path from "path";
import matter from "gray-matter";
import readingTime from "reading-time";

const POSTS_PATH = path.join(process.cwd(), "content/posts");

export function getPostBySlug(slug: string) {
  const fullPath = path.join(POSTS_PATH, `${slug}.mdx`);
  const source = fs.readFileSync(fullPath, "utf8");
  const { data, content } = matter(source);
  
  return {
    meta: data,
    content,
    readingTime: readingTime(content).text,
  };
}
```

<Callout type="tip">
  Always include a `readingTime` calculation. It's a small detail that significantly improves the user experience for long-form content.
</Callout>

## Step 3: Creating Custom Components

The beauty of MDX is using React components inside your Markdown. Let's create a custom `Callout` component.

```tsx title="components/mdx/Callout.tsx"
export function Callout({ children, type = "info" }) {
  const colors = {
    info: "bg-blue-50 border-blue-200",
    warning: "bg-amber-50 border-amber-200",
  };

  return (
    <div className={`p-4 border rounded-lg ${colors[type]}`}>
      {children}
    </div>
  );
}
```

## Step 4: The Rendering Engine

Now, let's create the dynamic route that renders the MDX content using the `MDXRemote` component from `next-mdx-remote/rsc`.

```tsx title="app/blog/[slug]/page.tsx"
import { MDXRemote } from 'next-mdx-remote/rsc';
import { getPostBySlug } from '@/lib/mdx';
import { Callout } from '@/components/mdx/Callout';

export default async function Page({ params }) {
  const post = getPostBySlug(params.slug);

  return (
    <article className="prose dark:prose-invert">
      <h1>{post.meta.title}</h1>
      <MDXRemote 
        source={post.content} 
        components={{ Callout }}
      />
    </article>
  );
}
```

## Step 5: Adding Syntax Highlighting

To make code blocks look professional, we'll use `rehype-pretty-code` with Shiki. It generates high-quality, pre-rendered syntax highlighting that works perfectly with Tailwind.

```typescript
// Add to your MDXRemote options
const options = {
  mdxOptions: {
    rehypePlugins: [
      [rehypePrettyCode, { theme: 'github-dark' }],
      rehypeSlug,
    ],
  },
};
```

Building a custom pipeline is more than just rendering text—it's about creating a developer experience for yourself.

TypeScript and React are a match made in heaven. But simply adding `.tsx` to your filenames isn't enough to get the full benefits of type safety. To truly "level up," you need to understand the patterns that allow TypeScript to catch bugs before you even save your file.

In this guide, we'll cover the essential patterns for modern React development.

## 1. Discriminated Unions for Component States

A common mistake is using separate boolean flags for loading, error, and data states. This leads to "impossible states" (e.g., `isLoading` and `isError` both being true).

### The Bad Way:
```typescript
interface Status {
  isLoading: boolean;
  isError: boolean;
  data?: string;
}
```

### The Better Way:
```typescript
type Status = 
  | { state: 'idle' }
  | { state: 'loading' }
  | { state: 'error'; error: string }
  | { state: 'success'; data: string };

function MyComponent({ status }: { status: Status }) {
  switch (status.state) {
    case 'loading': return <p>Loading...</p>;
    case 'error': return <p>Error: {status.error}</p>;
    case 'success': return <p>Data: {status.data}</p>;
    default: return null;
  }
}
```

## 2. Typing Component Props

Instead of manually typing `children: React.ReactNode`, use the built-in `PropsWithChildren` or `ComponentProps`.

```tsx
import { ComponentProps, PropsWithChildren } from 'react';

// For components with children
type ButtonProps = PropsWithChildren<{
  variant: 'primary' | 'secondary';
}>;

// For wrapping HTML elements
type InputProps = ComponentProps<'input'> & {
  label: string;
};
```

<Callout type="info">
  Using `ComponentProps<'button'>` allows your component to automatically inherit all standard HTML button attributes like `disabled`, `onClick`, and `type`.
</Callout>

## 3. Generic Components

Sometimes you want a component to work with any data type while maintaining type safety. This is where generics come in.

```tsx
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
}

function List<T>({ items, renderItem }: ListProps<T>) {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}

// Usage:
<List 
  items={['Apple', 'Banana']} 
  renderItem={(fruit) => <span>{fruit.toUpperCase()}</span>} 
/>
```

## 4. Typing Event Handlers

Don't use `any` for your event handlers. Use the proper generic types provided by React.

```tsx
function MyForm() {
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    console.log(e.target.value);
  };

  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    // submit logic
  };

  return (
    <form onSubmit={handleSubmit}>
      <input onChange={handleChange} />
    </form>
  );
}
```

## 5. Using `as const` for Type Narrowing

If you have a set of fixed strings or an object that shouldn't change, use `as const` to turn it into a literal type instead of a generic string.

```typescript
const CONFIG = {
  apiUrl: "https://api.example.com",
  timeout: 5000,
} as const;

// CONFIG.apiUrl is now the literal type "https://api.example.com"
// instead of just string.
```
